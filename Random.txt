CHAPTER 3: SQL IMPLEMENTATION
3.1 Introduction
This chapter focuses on the implementation of SQL queries based on various DBMS concepts such as aggregate functions, constraints, set operations, joins, subqueries, views, triggers, stored procedures, cursors, and exception handling. These queries are executed on the Mental Health Assessment Database to demonstrate data manipulation, retrieval, and automation.

3.2 Aggregate Functions
Aggregate functions are used to perform calculations on multiple rows of data and return a single value.

Query 1: Average Score
SELECT AVG(total_score) AS average_score FROM assessments;
Query 2: Maximum Score
SELECT MAX(total_score) AS max_score FROM assessments;
Query 3: Minimum Score
SELECT MIN(total_score) AS min_score FROM assessments;
Explanation:
These functions help in analyzing overall user performance in assessments.

3.3 Constraints
Constraints are used to enforce rules on data in a table.

Example: CHECK Constraint
INSERT INTO responses (assessment_id, question_id, score)
VALUES (1, 1, 25);
Explanation:
This query fails because the score exceeds the allowed range (1–20), ensuring data integrity.

3.4 Set Operations
Set operations combine results from multiple queries.

Example: UNION
SELECT user_id FROM users
UNION
SELECT user_id FROM assessments;
Explanation:
UNION removes duplicates and combines results from both tables.

Note:
MySQL does not support INTERSECT directly.

3.5 Joins
Joins are used to combine data from multiple tables based on a relationship.

Example: INNER JOIN
SELECT u.user_name, a.total_score
FROM users u
JOIN assessments a ON u.user_id = a.user_id;
Explanation:
This query retrieves user names along with their corresponding assessment scores.

3.6 Subqueries
A subquery is a query nested inside another query.

Example:
SELECT * FROM assessments
WHERE total_score > (
    SELECT AVG(total_score) FROM assessments
);
Explanation:
This retrieves assessments where the score is above average.

3.7 Views
A view is a virtual table based on the result of a SQL query.

Example:
CREATE VIEW user_scores AS
SELECT u.user_name, a.total_score
FROM users u
JOIN assessments a ON u.user_id = a.user_id;
SELECT * FROM user_scores;
Explanation:
Views simplify complex queries and improve security by restricting access.

3.8 Triggers
A trigger is a database object that automatically executes in response to certain events.

Example:
CREATE TRIGGER calculate_total_score
AFTER INSERT ON responses
FOR EACH ROW
BEGIN
    UPDATE assessments
    SET total_score = (
        SELECT SUM(score)
        FROM responses
        WHERE assessment_id = NEW.assessment_id
    )
    WHERE assessment_id = NEW.assessment_id;
END;
Explanation:
This trigger automatically updates the total_score whenever a new response is inserted, ensuring consistency.

3.9 Stored Procedure
A stored procedure is a precompiled collection of SQL statements stored in the database.

Example:
CREATE PROCEDURE get_result(IN aid INT)
BEGIN
    SELECT a.total_score, m.status_category
    FROM assessments a
    JOIN mental_health_reference m
    ON a.total_score BETWEEN m.score_range_min AND m.score_range_max
    WHERE a.assessment_id = aid;
END;
CALL get_result(1);
Explanation:
This procedure retrieves the mental health status based on the total score.

3.10 Cursor
A cursor is used to process query results row by row.

Example:
DECLARE cur CURSOR FOR SELECT user_name FROM users;
Explanation:
Cursors allow sequential processing of records.

3.11 Exception Handling
Exception handling is used to manage runtime errors.

Example:
DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = TRUE;
Explanation:
This handles the condition when no more rows are available in a cursor.

import mysql.connector

app = Flask(__name__)

        database="mental_health_tracker"
    )

# Home page

    return render_template('index.html')

@app.route('/questions')
def questions():
    return render_template('Questionnaire.html')

# Submit form
@app.route('/submit', methods=['POST'])
def submit():
    name = request.form['user_name']
    email = request.form['user_email']

    conn = get_db_connection()
    cursor = conn.cursor()

    # Insert user
    cursor.execute(
        "INSERT INTO users (user_name, user_email) VALUES (%s, %s)",
        (name, email)
    )
    user_id = cursor.lastrowid

    # Create assessment
    cursor.execute(
        "INSERT INTO assessments (user_id) VALUES (%s)",
        (user_id,)
    )
    assessment_id = cursor.lastrowid

    # Insert responses (assuming 10 questions)
    total_score = 0
    for i in range(1, 11):
        score = int(request.form[f'q{i}'])
        total_score += score

        cursor.execute(
            "INSERT INTO responses (assessment_id, question_id, score) VALUES (%s, %s, %s)",
            (assessment_id, i, score)
        )

    conn.commit()

    # Fetch result
    cursor.execute("""
        SELECT status_category 
        FROM mental_health_reference
        WHERE %s BETWEEN score_range_min AND score_range_max
    """, (total_score,))
    
    result = cursor.fetchone()[0]

    conn.close()

    return render_template('result.html', score=total_score, result=result)

if __name__ == '__main__':
    app.run(debug=True)
